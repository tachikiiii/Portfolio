Izakaya Games

飲み会で遊びながら
支払いの計算ができるゲームアプリ。
酔った状態でも簡単に操作でき、
負けた人が支払いを８割負担するなどの設定が可能。

開発の経緯

居酒屋経営をする知人から
「テーブルで遊べるWebアプリを作りたい」
との相談を受け、お会計を絡めたゲームを提案。
JavaScript初学者と「素のJavaScript」で
ペアプログラミングを行い、
直感的なUIと割り勘ロジックを融合した
アプリを開発しました。

使用技術

  • HTML / CSS
  • JavaScript

ペアプログラミング

Github を活用してページごとに開発を分担、定期的なオンラインミーティングで進捗と課題を共有しました。 各ミーティング後、宿題としてタスクを割り当て、次回までにコードレビューとテストを完了することで、 開発プロセスの透明性と品質を維持しました。 さらに、ライブコーディングセッションも実施し、リアルタイムでの問題解決と知識共有に努めました。

こだわったポイント

  • 　酔った状態でも操作しやすい直感的なUI
  • 　ゲーム結果に応じた割り勘ロジック

苦労した点

JavaScript の実装方法には多様性があるため、
2人で分担してコードを作成した結果、
プロジェクト全体として統一感に欠ける状態になりました。
そのため、コードスタイルを揃えるために労力を要しました。

改善点

ゲーム

ゲームは現在１種類のみだが、数種類から選べるようにする。

ペアプログラミング

ページごとに分担して行なったが、
本来のペアプログラミングの役割である
「ドライバー」と「ナビゲーター」に分かれてプロジェクトに取り組み、
プロジェクト全体の把握をしやすくする。

シェイクカウンター

スマホを実際に振った回数と検出カウントにズレが生じている。
パラメータの調整は試みたが、
複数軸の加速度データを組み合わせた解析を行い、
誤検出を減らす検出ロジックの検討をする。
